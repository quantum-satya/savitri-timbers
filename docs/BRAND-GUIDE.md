# Purpose

The Savitri Timbers Design System defines the visual, functional and interaction standards for the company's digital presence.

It exists to ensure that every page, component, feature and future enhancement communicates a consistent brand identity while maintaining high standards of usability, accessibility, performance and maintainability.

This document serves as the master index for the complete design system.

Detailed standards are maintained in the individual documents located in the `docs/brand/` directory.

---

# Design Philosophy

The Savitri Timbers website should inspire confidence before it generates enquiries.

Visitors should immediately recognize the company as:

- Professional
- Premium
- Reliable
- Trustworthy
- Experienced
- Modern
- Customer-focused

The website should never appear cluttered, overly decorative or sales-driven.

Every design decision should reinforce long-term business relationships rather than short-term marketing.

---

# Design Principles

Every page should follow these principles.

## Clarity over complexity

Keep layouts clean and easy to understand.

---

## Trust before marketing

Professional presentation builds confidence.

Avoid exaggerated claims.

---

## Simplicity wins

Prefer fewer, better-designed components over many competing elements.

---

## Performance matters

Fast websites create better user experiences.

Every component should justify its existence.

---

## Consistency over creativity

Reusing established patterns is preferred over introducing new ones.

---

## Mobile first

Every page should work beautifully on mobile devices before desktop enhancements are considered.

# Design Goals

Every page should achieve one or more of the following objectives.

- Build trust
- Strengthen brand identity
- Generate enquiries
- Present products clearly
- Improve user experience
- Showcase current business capabilities
- Encourage long-term customer relationships

# User Experience Principles

The user should always know:

- Where they are
- What the company offers
- What to do next
- How to contact the company

Every page should have a clear primary call to action.

---

# Accessibility

The design system follows accessibility best practices.

Future chapters define standards for:

- Semantic HTML
- Keyboard navigation
- Color contrast
- Image alt text
- Accessible forms

Accessibility is considered part of quality.

---

# Performance Standards

The website should remain:

- Lightweight
- Fast
- Responsive
- Secure
- Easy to maintain

Performance should never be sacrificed for unnecessary visual effects.

---

# Responsive Design

Every component must support:

- Desktop
- Laptop
- Tablet
- Mobile

No component is considered complete until responsive behavior has been verified.

---

# AI-Assisted Development

The Savitri Timbers project actively uses AI during development.

AI tools should:

- Follow this design system.
- Reuse existing components.
- Preserve consistency.
- Avoid unnecessary redesigns.
- Respect business rules documented in PROJECT.md.

AI should assist design—not redefine it.
